Script - uglify - Pack centauri-bash-lib sources, create packaged script
Usage:
(1) uglify option... -|module... (2) uglify [-N] [-X] --output packed script [module...]
This tool strips comments and extra spaces from bash sources. Modules with lots of comments load a little bit faster when stripped. The proxy of centauri-bash-lib and the embed function load stripped sources if they are up to date:
Usage (1) creates packed module.p files. The recommended usage is:
$ uglify - # use all modules
To generate module.p files for some modules only use:
$ uglify module... # where <module> is a file name
Usage (2) packs a script and the used modules (stripped) into a single file. Modules that are not auto-detected can be specified as arguments.
Options:
-d --dryrun show what would be done, make no changes -f --force override prudent default behaviours -h --help show this text -n --no do not prompt for confirmation, assume 'no' -o --output file Open file for output -q --quiet report errors only -v --verbose be verbose (use twice for debug) -y --yes do not prompt for confirmation, assume 'yes'
--embed name use name instead of script name --info print full path and version --pager prog pipe output thru pager, use 'none' to disable --trace trace actions() or run() --wait prompt before exit
-C --config file (packaging) use a configuration file -N --nodeps (packaging) ignore "embed -[cmq]" in library modules -X --extractor (packaging) add self-extractor (_centauri_bash_xtr)
Arguments:
module a module name (3 letter or full name) packed generated packed script script a script to be packaged with centauri-bash-lib
Limits:
uglify does not understand full bash syntax. Here some of the limits:
- multi-line strings are not supported - inline documents must use "!EOF" as senteniel - lines with nested strings will not be optimized
Dependencies:
Form (2) tries to auto-detect module dendencies (see --nodeps). To hide a dependency use 'embed -r ...' with '-r' being the 1st option. To add a dependency without actually initializing the module use 'embed -c ...'. Custom modules are always ignored but can be added using command args.
