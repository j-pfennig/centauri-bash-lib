Script - uglify - Pack centauri-bash-lib sources, create packaged script
Usage:
(1) uglify option... -|module... (2) uglify [-N] [-X] --output packed script [module...]
This tool removes comments and extra spaces from bash sources. Modules with large comments load a little bit faster when 'uglified'. The proxy of centauri-bash-lib and the embed function load stripped files if they are up to date (newer than the source).
A simalar pre-processing can be done to speed up centaurihelp. This is implemented by removing the script code and saving only documentation.
Usage (1) creates packed module.p and/or <module.d> files:
$ uglify -U # use all modules, uglify only $ uglify -D -U # also process documentation $ uglify -D # only process documentation
To generate module.p or .d files for some modules only use:
$ uglify module... # where <module> is a file name $ uglify -D module...
Usage (2) packs a script and the used modules (stripped) into a single file. Modules that are not auto-detected can be specified as arguments. The centauribuild tool runs uglify for building stand-alone scripts.
Options:
-d --dryrun show what would be done, make no changes -f --force override prudent default behaviours -h --help show this text -n --no do not prompt for confirmation, assume 'no' -o --output file Open file for output -q --quiet report errors only -v --verbose be verbose (use twice for debug) -y --yes do not prompt for confirmation, assume 'yes'
-C --config file (packaging) use a configuration file -N --nodeps (packaging) ignore "embed -[cmq]" in library modules -X --extractor (packaging) add self-extractor (_centauri_bash_xtr)
-D --docu (pre-processing) create *.d files for centaurihelp -U --uglify (pre-processing) create *.p files for faster loading
Option --uglify is implied if no other option is specified
Arguments:
module a module name (3 letter or full name) file configuration file, see centaurilauncher packed generated packed script script a script to be packaged with centauri-bash-lib
Limits:
uglify does not understand full bash syntax. Here some of the limits:
- multi-line strings are not supported - inline documents must use "!EOF" as senteniel - lines with nested strings will not be optimized
Dependencies:
Form (2) tries to auto-detect module dependencies (see --nodeps). To hide a dependency use 'embed -r ...' with '-r' being the 1st option. To add a dependency without actually initializing the module use 'embed -c ...'. Custom modules are always ignored but can be added using command args.
