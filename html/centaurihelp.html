Script - centaurihelp - Dokumentationsgenerator für centauri-bash-lib und centauritools
Aufruf:
centaurihelp [-A] [-C] [-I] [-M] [-R] [-H folder] [-N] [-U file] ... centaurihelp -T [-H folder] [-N] [-U file] ... centaurihelp -G what [-H folder] ... centaurihelp -D ...
Dokumentationsgenerator für centauri-bash-lib und centauritools.
Dieses Skript kann Dokumentation im HTML Format erzeugen oder kann aus der Kommandozeile so wie das Linux "man" Programm benutzt werden.
Optionen:
-d --dryrun Zeigen was getan wird, keine Änderungen vornehmen -f --force Vorsichtsmassnahmen ausser Acht lassen -h --help diesen Text ausgeben -i --include expr Nur zum Ausdruck passende Dinge benutzen -n --no Nicht nach Bestätigung fragen, 'nein' annehmen -q --quiet Nur Fehler melden -v --verbose Viel Ausgabe (zweimal zum Testen) -x --exclude expr Zum Ausdruck passende Dinge ausschliessen -y --yes Nicht nach Bestätigung fragen, 'ja' annehmen
--embed name Benutze name anstelle des Script Namens --info Vollen Pfad und Version ausgeben --pager prog Programm für seitenweise Ausgabe, 'none' für keines --trace Trace actions() oder run() --wait Nachfrage vor Beenden
-D --delete Lösche Caches -G --generate what Erzeuge HTML Dokumentation -H --html folder HTML Ausgabe nach Verzeichnis (impliziert --all) --file Nur Text: Dateinamen anzeigen (impliziert --lines) --lines Nur Text: Zeilennummer ausgeben -L --library Zeige 'library' Dokumentation an (Vorgabe) -T --tools Zeige 'tools' Dokumentation an -U --use file Verarbeite einzelne Datei, "-U ." für 'centaurihelp'
-A --all Aktualisiere oder erstelle index/modules/content/referenz -C --content Inhaltsverzeichnis erstellen -I --index Nur HTML: erstelle 'index.html' -M --modules Tabelle von Modulen erstellen -R --reference Referez von Funktionen/Punkten erstellen
Argumente:
what Auswahl: all | tools | library | dummy folder Ausgabeverzeichnis (- für /dev/stdout, + für Abfrage) file Irgendein Tool oder eine Library Datei
Filter:
Alle weiteren Kommandozeilenargumente sind Ausgabefilter. Die Zahl von Filtern ist beliebig, Filter wirken kumulativ. Es gibt drei Selektoren um Ausgabe auf Funktionen '-', Topics '+' oder Module '=' zu beschränken.
Beispiele:
# Erzeugte Ausgabe auswählen centaurihelp --content # output table-of-content centaurihelp --all # also output index/modules/content/reference centaurihelp -i ext # only output _centauri_bash_ext documentation centaurihelp folder att # filter for '*folder*' or '*att*' centaurihelp -M move # in which module is 'move' centaurihelp -C -i _sys # what is in module _centauri_bash_sys centaurihelp - sys # restrict output to functions
# Dokumentation als HTML erstellen (das Zielverzeichnis ist 'doc') centaurihelp --delete --all --html doc
# Dokumentation als Datei erstellen/ansehen (--force für ANSI Escapes) centaurihelp --all --force > doc.txt less -FRX doc.txt
# Erzeuge HTML Dokumentation centaurihelp --generate=tools -H mydir # tool docs to custom location centaurihelp --delete --generate=all # clear cache, docs to default
