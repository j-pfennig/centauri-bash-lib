Script - autoload - Create the autoload module _centauri_bash_dyn
Usage:
autoload option... argument...
Create the autoload module _centauri_bash_dyn.
Options:
-d --dryrun show what would be done, make no changes -h --help show this text -n --no do not prompt for confirmation, assume 'no' -q --quiet report errors only -v --verbose be verbose (use twice for debug) -y --yes do not prompt for confirmation, assume 'yes'
--embed name use name instead of script name --info print full path and version --pager prog pipe output thru pager, use 'none' to disable --trace trace actions() or run() --wait prompt before exit
-B --builtin enable loadable bash builtins (implies --dynamic) -D --dynamic enable dynamic module loading -I --ignore name ignore loadable, repeatable (implies --builtin) -N --nocustom exclude user custom modules (implies --dynamic)
Remark:
Dynamic module loading has no flaws and should always be enabled. Loadable buildins usually work fine but are often refered to as 'examples'. autoload has a builtin list of loadables that are supported by the library. autoload only enables a builtin if loading and a functionality test succeed.
Examples:
The recommended usage is:
$ autoload --builtin # include all modules, use all loadables $ autoload --dynamic # include all modules, don't use loadables
To generate autoload for selected modules (use no loadables):
$ autoload -D mod... # where <mod> is a 3 letter name
Disable some loadable builtins, include all modules ...
$ autoload -I realpath -I ln
